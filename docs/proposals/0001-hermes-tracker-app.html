Hermes Tracker APP

Proposal · 2026-06-27 · David (claude-bridge maintainer)

Discord thread = 訊息流,難掌握 overall project progress。 本提案將 Hermes 由 fire-and-forget autonomous PM 變成 supervised planning — Claude 提出 plan,你睇完先批。

Web · PWA · Tauri?

Pain point

Discord 適合 command-driven workflow,

  • ❌ 冇全局視角 — 要逐個 thread 入去先睇到 plan / journal / cost
  • ❌ 冇 timeline visualization — Discord 係 linear chat,journal 唔容易睇
  • ❌ 冇 friendly doc review — plan.md 係 raw markdown,要打開 VS Code 讀
  • ❌ 冇 structured Todo summary — 冇地方睇「邊啲 task 做咗 / 邊啲 pending」
  • ❌ 冇 push notification — 重要事件(project done / fail / cost spike)冇主動通知

4 capabilities

A · Read

Summarized project progress — structured Todo view + AI summary + live updates

B · Write

Add Todo interactively — quick append OR multi-turn Claude conversation

C · Approve

Plan/Doc approval workflow — review plan.md as HTML before execution starts

D · Cross-platform

Web UI (responsive) + PWA install on mobile + optional Tauri desktop

Integration tests failed │ │ on first attempt (DB race); retry in progress. │ │ Documentation is the next planned task." │ └─────────────────────────────────────────────────────┘

Implementation

  • Static view 從 state.plan + state.costUsd + state.timer 直接 render(冇 AI)
  • AI Summary 用 haiku summarizer — journal.log recent N entries → 2-3 sentences status
  • New summarizeProject(state) helper(unit-testable)
  • Live update via SSE — plan task status 即時 refresh

B · Write — Add Todo (Interactive)

Mode A: Quick add

[+ Add Todo] Title: implement dark mode toggle Description: should respect system preference [Cancel] [Submit →]

Flow:

  1. APP → POST /api/projects/:id/todos {title, description}
  2. Bot appends 到 state.plan 直接
  3. Save state, push SSE event "new task added"
  4. Orchestrator 下一個 judge pass 自然 incorporate

Mode B: Interactive w/ Claude

[+ Add Todo (with Claude)] Goal: implement dark mode 💬 Claude: "What framework are you using?" 📝 You: React + Tailwind 💬 Claude: "Should it persist user choice across sessions?" 📝 You: yes 💬 Claude: "Got it. Proposed task:" ───────────────────── t7: Add dark mode toggle ─ React + Tailwind, persists in localStorage ─ Auto-detect system preference ─ Add manual override in settings ───────────────────── [Cancel] [Refine] [✓ Approve & Add]

Flow:

  1. APP → POST /api/projects/:id/todos/interactive {goal} 開新 conversation
  2. Bot spawn Claude sub-agent(haiku, permissionMode="plan")用 MCP tools ↔ APP
  3. APP ↔ Claude 來回直到 Claude 提出 proposed task spec
  4. Approve → 寫入 plan
  5. 加 budget timer 防止無限 loop

C · Plan/Doc approval workflow

Phase 0 — Hermes 寫 doc

User: /project start "build a CLI todo app"
  ↓
Planner LLM generates:
  - state.plan (array of tasks)
  - plan.md (human-readable, currently ignored by APP)
  ↓
Hermes saves both, transitions status → "awaiting_approval"

Phase 1 — APP 推送通知

┌─ Notifications ────────────────────┐ │ 📋 abc12345 needs plan review │ │ "build a CLI todo app" │ │ 7 tasks proposed, est. $1.20 │ │ [Review plan →] │ └─────────────────────────────────────┘

SSE push: event: approval-needed

Phase 2 — Plan review UI(HTML rendered)

┌─ Plan: build a CLI todo app ──────────────────────┐ │ Summary: A 7-task plan to scaffold a Node.js │ │ CLI with persistent storage and JSON output. │ │ │ │ 📋 Plan │ │ ───────────────────────── │ │ ## Tasks │ │ ▢ t1: Initialize Node.js project + deps │ │ ▢ t2: Define Todo interface (TS) │ │ ▢ t3: Implement file-based persistence │ │ ▢ t4: Add commands: add / list / done / rm │ │ ▢ t5: Add JSON output flag │ │ ▢ t6: Write unit tests │ │ ▢ t7: Write README + smoke test │ │ │ │ ⚠️ Risks │ │ - File locking not addressed (single-user only) │ │ - No migration story if schema changes │ │ │ │ 💰 Estimated cost: $1.20 (6-8 min) │ │ ⏱ Estimated wall: 30m │ │ │ │ 💬 Comments (0) ──── │ │ [Add a comment...] │ │ │ │ [Reject] [Request changes] [✏️ Edit tasks] [✓ Approve & run] └──────────────────────────────────────────────────┘

Markdown → HTML: server-side 用 marked + highlight.js,cached by file mtime。

Phase 3 — User actions

ActionResult
✓ ApproveHermes awaiting_approval → executing,orchestrator 開工
✏️ Edit tasksdrag-drop reorder / delete / add → save → approve
Request changesFree-form comment → planner refines → v2 plan
✗ RejectStatus failed, killedReason="user_rejected_plan"

D · Cross-platform

  • Web UI — Responsive HTML, mobile-first
  • PWA install — Mobile 用戶「加到主畫面」= standalone APP,離線 cache static assets
  • Desktop — Browser tab (MVP),或者 Tauri wrapper (optional,~10MB binary)
唔推薦: React Native / Flutter / Native — over-engineering 對 single-user tool。 Web + PWA 已經 cover 80% use case。

Phased rollout

  1. P1 · Backend API
    Projects list/detail, journal SSE, stats endpoints
    ~1 day
  2. P2 · Read UI
    Dashboard, project detail, progress, AI summary
    ~2-3 days
  3. P2.5 · Doc Approval
    md → HTML, awaiting_approval state machine, comments
    ~1-2 days
  4. P3 · Add Todo
    Quick add + interactive w/ Claude (multi-turn chat)
    ~2 days
  5. P4 · PWA
    manifest.json + service worker, mobile install
    ~0.5 day
  6. P5 · Tauri (optional)
    Desktop wrapper,~10MB binary,system tray
    ~2-3 days
MVP (P1-P3): ~7-8 working days。
Full (P1-P5): ~10-12 days including Tauri wrapper。

Data model changes

state.json 新加兩個 fields:

interface ProjectState {
  // ... existing fields ...
  status: ... | "awaiting_approval";  // NEW
  approval?: ApprovalRequest;          // NEW
}

interface ApprovalRequest {
  docPath: string;          // e.g. "plan.md"
  docMtime: number;
  requestedAt: string;
  reviewerNotes?: string;
  approvedAt?: string;
  rejectedAt?: string;
  rejectionReason?: string;
}
