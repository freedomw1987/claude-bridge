## 🔥 最高 ROI 隱藏問題 (新發現)

### N3. **`activeProcessCount()` 唔 count SDK runs** — 真係 concurrency cap bug
**位置**: `src/cleanup.ts:20` 只 count CLI PIDs; `src/agent/sdkRunner.ts:84` 用獨立 `activeRuns` map。
**問題**: `MAX_CONCURRENT_CONTAINERS=5` 默認只 cap CLI runs。**SDK runs 唔受限制**, 10 個 thread 同時用 SDK 可以 spawn 10 個 query(), 全部都爆 cap 而冇任何 warn。
**修**:
```ts
// cleanup.ts
export function activeProcessCount(): number {
  return activeProcesses.size + activeSdkRunCount();  // 統一
}
```
**Risk**: Medium (改 hot path, 但 logic 簡單)
**Effort**: 1h
**ROI**: **極高** (生產環境真係可能 over-spawn)

---

### Q3. **`env: { ...process.env }` 過寬** — Security
**位置**: `src/agent/runner.ts:181`
**問題**: 將 `~/.aws/credentials` / `GITHUB_TOKEN` / `GH_TOKEN` / `NPM_TOKEN` 等敏感 env 傳晒畀 `claude` subprocess。
**修**:
```ts
const SAFE_ENV_KEYS = [
  "PATH", "HOME", "USER", "SHELL", "LANG", "LC_ALL", "TZ",
  "ANTHROPIC_API_KEY", "ANTHROPIC_AUTH_TOKEN",
  "CLAUDE_CODE_USE_BEDROCK", "CLAUDE_CODE_USE_VERTEX",
  "DISPLAY", "TMPDIR",
];
env: Object.fromEntries(
  SAFE_ENV_KEYS.map(k => [k, process.env[k]]).filter(([,v]) => v != null)
)
```
**Risk**: Low (新增 feature, 唔影響現有 runs)
**Effort**: 30min
**ROI**: 高 (security baseline)

### N6. **Hermes judge `needs_more` 唔 check cost cap** — Cost blow-up risk
**位置**: `src/hermes/orchestrator.ts:208-223`
```ts
if (verdict.verdict === "needs_more" && verdict.nextTasks?.length > 0) {
  state.plan.push(...verdict.nextTasks);  // 無止境 add tasks
  state.status = "executing";
  // ← 呢度冇 check costUsd 會唔會爆 maxCostUsd
}
```
**問題**: Judge 可以一直 say `needs_more`, 加 task, 跑到 `maxIterations` cap 先停。但 `HERMES_MAX_COST_USD` 形同虛設 (要等下次 run 失敗先 trigger)。
**修**:
```ts
if (verdict.verdict === "needs_more" && verdict.nextTasks?.length > 0) {
  const estimatedCost = state.costUsd + (verdict.nextTasks.length * AVG_TASK_COST);
  if (estimatedCost > state.config.maxCostUsd) {
    log.warn("hermes: skipping judge-added tasks (cost cap)", { ... });
    state.status = "failed";
    state.killedReason = "cost_cap";
    return;
  }
  state.plan.push(...verdict.nextTasks);
  ...
}
```
**Risk**: Low (防禦性 fix)
**Effort**: 2h
**ROI**: 中 (cost safety, 用戶可能 encounter)

### F2. **Approval gate** — 真係 zero 防守
**位置**: `src/agent/sdkRunner.ts:198-208`
```ts
if (toolName.startsWith("mcp__discord-bridge__")) return { behavior: "allow" };
return { behavior: "allow" };  // 全部都 allow
```
**問題**: `permissionMode: "bypassPermissions"` 已經 disable 咗 CC 嘅 permission prompts, 加埋 canUseTool 全部 allow → **user typo `rm -rf /` 就直去**。
**修**:
```ts
if (toolName === "Bash") {
  const cmd = String(_input.command ?? "");
  if (DANGEROUS_PATTERNS.some(p => p.test(cmd))) {
    // Pause + post "⏸️ approval needed" embed + wait for ✅/❌ reaction
    const approved = await requestUserApproval(thread, cmd);
    if (!approved) return { behavior: "deny", message: "user rejected" };
  }
}
return { behavior: "allow" };
```
**Risk**: Medium (新 feature, UX 要小心)
**Effort**: 2d
**ROI**: **極高** (safety critical)
